9.5 Solid Section

https:://concrete-fem.com

A section starting with *SOLID SECTION defines a set of truss elements (T1D2,T2D2,T3D3,TAX2,TAX3) or corresponding embedded types forming rebars, or spring elements (S1D2) or 2D-continuum elements (CPE3/4,CPS3/4,CAX4). All elements within a section have the same properties.
Multiple section definitions are allowed if required to define a complete model.
The section keyword *SOLID SECTION has as options

ELSET=<name>

refers to a name of a set of elements (mandatory)
 

(the same name also has to be defined within an element section)

MATERIAL=<name>

refers to a material name defining the material of the elements (mandatory)
 

(the same name also has to be defined within a material section)

NREBAR=<float>

connected to a set of embedded elements of type T2D2I,T2D3I,TAX2I (optional)
 

is ignored for other element types

 

specifies the number of rebars per unit width – with length unit as is used in data set – to consider bonding

 

if not specified default is with thickness of underlying CP*,CAX*-elements

 

explicit definition of NREBAR is recommended for TAX2I-elements



The data line describes the following data


with

width in case of elements CP*

 

cross section area in case of elements T1D2,T2D2,T3D2,T3D3 and corresponding embedded types
 

height in case of elements TAX2, TAX3 whereby width is ruled by circumferential radius – these elements behave like a sheet

 

spring stiffness (force/length) in case of elements S1D2 (used only with material type *ELASTIC)


is ignored in case of elements CAX*. A unit width is used instead.
In case NREBAR is specified with elements T2D2I,T2D3I the cross section area refers to a single rebar.
Take care to use the same unit as for all other length quantities.

The section keyword *SOLID SECTION has no own sub-keywords.
